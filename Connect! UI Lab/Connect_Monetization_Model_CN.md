# Connect! — 变现模型(商业模式)

> **状态:** 供审阅的工作草稿,**尚未**合并进 CLAUDE.md。
> **定位:** 配合"Community-First / 跑步先行"的转向阅读,搭配 §1(Product)与 Stage 计划。
> **一句话:** Connect! 不是抽佣生意,而是一个**受众 / 社区层**。跑者免费参与;**付费的是品牌、雇主和认真的组织者**——他们花钱来触达和服务这群人。

---

## 0. 核心原则(先读这条)

**把"你获取谁"和"你向谁收费"分开。**

- **你获取谁:** 跑者 / 会员 —— *永远免费,这是刻意设计。*
- **谁向你付费:** 任何有预算想**触达**或**服务**这群人的一方。

跑者是你的**库存和受众**,不是你的客户。最经典的错误,就是想让受众掏钱——在一个免费的运动里,这会同时杀死社区和增长。免费参与是**引擎**:它让受众变大,而这恰恰是受众对付钱方有价值的原因。这是**媒体 / 社区**的玩法,不是 marketplace 抽佣的玩法。

**为什么这比 Playtomic 是一个不同(且更肥)的游戏**

| | Playtomic | Connect! |
|---|---|---|
| 靠什么变现 | **可预订球场**上的交易(每单佣金 + 球场 SaaS) | 一个富裕活跃社群的**注意力、归属与数据** |
| 毛利形态 | 薄、按单计(约 €0.20/单) | 经常性 B2B 合同(四到五位数),轻资产 |
| 资产依赖 | 需要球场 / 场地库存 | 无 —— 搭在已存在的线下社群上 |
| 在跑步里能触达吗 | 不能(没球场 = 没东西可订) | 能 —— 跑步就是楔子 |

Playtomic 占住的是"订场交易层"。它**结构上无法**从一场免费的 Corniche 晨跑里变现——那里根本没有可抽成的东西。而那个变现层,本来也喂不饱一个**跑步**受众。Connect! 变现的,是 Playtomic 根本不经营的那一层。

## 1. 谁免费 vs 谁付费

| 群体 | 免费? | 角色 |
|---|---|---|
| 跑者 / 会员 | 免费 | 受众(库存) |
| 随手办活动的组织者 | 免费 | 供给 / 获客渠道 |
| **品牌与赞助商** | **付费** | 想触达受众 |
| **雇主(企业健康计划)** | **付费** | 想服务自己员工 |
| **认真的组织者** | **付费(SaaS)** | 想要专业办活动工具 |
| 买高端体验的会员 | **按次付费** | 为**体验**付费,绝不为"参与"付费 |

## 2. 收入线(按近期可行度排序)

### 线 1 —— 品牌赞助与合作  *(主力、经常性 B2B)*
**谁付费:** 运动品牌(New Balance 已经在多哈办跑团;Nike、On、Hoka 都想要 run-club 存在感)、精品咖啡与 F&B(咖啡闭环)、银行与电信(QNB、Ooredoo —— 巨额"活跃生活" CSR 预算,挂钩 Qatar National Vision 2030 / 健康议程)、地产与目的地(Msheireb、The Pearl、Lusail —— 要客流 + 生活方式联想)。
**他们买什么:** 联名跑步系列、"官方主场"地位、活动现场存在感,以及**受众数据 / ROI 报告**。
**为什么 Playtomic 占不了:** Playtomic 卖订场,不卖非球场运动的社群赞助。
**对你的要求:** 受众规模 + 可信度、品牌活动页,以及**赞助报告生成器**(已在你的功能表里——它是这条线的**销售武器**)。
**阶段:** Phase 1(本地 / 咖啡 / 运动品牌)→ Phase 2(银行、电信、区域级)。

### 线 2 —— 企业健康计划 / B2B 员工项目  *(强力的早期线——不依赖规模)*
**谁付费:** 公司,为员工跑团、步数 / 里程挑战、跨公司活动、健康报告买单。
**为什么是聪明的早期楔子:** 一个企业单子只需要**一个决策人(HR / 健康负责人)**,而且**只需要这家公司自己的员工**——**不需要全城受众**。所以你能在还没起量时就先有收入。经常性预算。海湾企业有真实的健康预算。
**对你的要求:** 公司 / 团队分组、私有公司跑团、挑战 / 排行榜,以及给赞助方的**管理后台 + 参与报告**。
**阶段:** Phase 1(可以是你的**第一条**付费线)。

### 线 3 —— 付费高端体验与活动
**谁付费:** 会员,按体验付。免费周跑是漏斗;**付费体验**才变现。
**例子:** 沙漠 / 日出跑 + 早餐、Ladies-only 周末 retreat、**斋月夜跑系列**、赛事日 package、联名 merch。
**对你的要求:** 活动页 + 走**外部支付通道**售票(见"风险"——不要自建 / 沉淀资金)。
**阶段:** Phase 1–2。

### 线 4 —— 组织者 SaaS(付费高级版)  *("武装 dealer")*
**谁付费:** 办收费活动 / 训练营的认真组织者。
**他们买什么:** 报名 + 候补、**对账**、**赞助报告生成器**、会员 CRM、品牌活动页。
**为什么 Playtomic 占不了:** Playtomic Manager 服务球拍**球场**;没人服务社群 / 耐力**组织者**。
**阶段:** Phase 2。

### 线 5 —— 更远 / 可选
跑步教练与训练计划 marketplace(重新引入支付通道——先放着);装备 affiliate / 电商(低毛利——次要)。

## 3. 分阶段推进(收入绑定规模)

| 阶段 | 时间*  | 目标 | 收入 |
|---|---|---|---|
| **0 —— Doha:堆受众** | ~0–6 月 | 拿下 1–2 个 hero 组织者;跑通 playbook;咖啡闭环做**易货**(客流 + 装机驱动) | **刻意 ≈ $0** |
| **1 —— Doha:首笔收入** | ~6–18 月 | 企业健康单;咖啡"官方主场"转付费(报告促成);首个本地 / 运动品牌赞助;付费高端活动;给少数认真组织者的 SaaS | 首批经常性 B2B |
| **2 —— GCC(UAE / 沙特):开始赚钱** | ~18 月+ | 区域品牌单、多城企业项目、规模化活动 / 赞助 package | **在这里变成真生意** |

\*对单人创始人仅作示意;每个阶段**用"是否被验证"来卡,而不是日历。**

**与扩张的绑定(重要):** 单城 Doha 的赞助盘子薄。模型在 **GCC 规模**下才赚钱,因为品牌和企业买的是**全区域触达**。变现和 Qatar → UAE → 沙特 的扩张是**同一个计划,不是两个**。(注意:社区产品在每座城市都要**重新冷启动**——你复制的是 *playbook*,不是受众。)

## 4. 示意经济模型(只看形状——用验证过的数字替换)

> ⚠️ 这些是**用来展示模型形状的占位假设**,不是调研出来的市场价。**在相信这个模型之前,用一笔真实成交去验证每一个数字。**

| 收入线 | 示意单位 | 示意价格(QAR) |
|---|---|---|
| 企业健康计划 | 每公司 / 月 | 3,000 – 8,000 |
| 咖啡"官方主场" | 每店 / 月 | 1,000 – 3,000(早期易货) |
| 品牌联名跑系列 | 每季度系列 | 15,000 – 40,000 |
| 组织者 SaaS | 每组织者 / 月 | 100 – 300 |
| 高端活动 | 每张票(你抽 15–25%) | 50 – 150 |

*示意的 Phase-1 Doha 收入栈:* 4 个企业项目 + 3 个咖啡单 + 1 个季度赞助 + 10 个 SaaS 组织者 → 一个不大但真实的经常性底盘。**重点是形状,不是这些数字**——把你第一笔验证过的成交填进去。

## 5. 留存 —— 对应到每一类客户

*(直接回答"这个 app 没有吸引和留住客户的功能"——它有,而且是两边都有。)*

| 客户 | 靠什么留住 |
|---|---|
| **跑者(受众)** | 社区 + 身份 / streak / 数据 + **权益网络(线下真实价值 = Strava 给不了的护城河)** + 跨 club 发现 + 归属感 |
| **赞助商** | **ROI 报告**(报告生成器)+ **"官方主场"独家性** + 持续的活动现场存在 |
| **企业** | 管理后台 + 挑战 / 排行榜 + 参与度报告 |
| **组织者** | SaaS 工具本身(工作流切换成本) |

## 6. 这对产品的要求(→ 对 CLAUDE.md 的影响)

- **赞助报告生成器** —— 聚合的受众指标与画像。**PDPPL 合规:仅聚合 / 经授权;绝不卖个人数据。**
- **品牌活动页 + club↔赞助商绑定**("[某跑团] 官方主场")。
- **企业 / 团队层** —— 私有公司跑团、挑战、赞助方管理后台 + 报告。
- **付费活动** —— 接**外部售票 / 支付服务商**;**不要**自建或沉淀资金(让你远离那个你刻意躲开的合规负担)。
- **组织者 SaaS 付费墙层** —— 把对账、报告生成器、CRM 收进付费档。
- **留存原语** —— 权益网络、会员身份 / 数据、跨 club 发现。

## 7. 诚实的风险

- **延迟变现。** Phase 0 ≈ $0。靠低 burn 撑(不碰支付、单人)——但没有早期现金,需要耐心 / 跑道。
- **先有鸡还是先有蛋。** 赞助要受众规模。**对策:** 用**企业健康计划**(不依赖规模)+ **咖啡易货**先开局。
- **PDPPL(已在执法;罚款 QAR 100 万–500 万)。** 任何数据变现必须**聚合 + 授权**;报告必须隐私干净。
- **支付通道。** 付费活动 / 教练会重新引入合规——**外包通道**,别沉淀资金。
- **海湾赞助是关系驱动、且收入不均匀。** 早期集中度风险(靠 1–2 个大赞助 = 脆弱)。
- **单城天花板。** 不扩张到 GCC 可能永远做不大——而每座新城都是一次全新的冷启动。

---

## 8. 一句话总结

你在做的,是**"海湾活跃生活方式的受众层"**——一个品牌垂涎、难触达的高价值社群——然后像媒体和社区那样变现:**品牌和雇主付钱来触达和服务这群人;组织者付钱买工具;会员只为高端体验付费;所有人免费跑。** 这和 Playtomic 那种薄薄的按单佣金是**不同的品类**——更肥的毛利、轻资产,而且结构上在 Playtomic 够不着的地方。你早就设计好的那些零件(赞助报告生成器、咖啡闭环、分享卡)**不是抽佣失败后的备胎——它们就是主模型。**
